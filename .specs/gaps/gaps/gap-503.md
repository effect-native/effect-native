# Gap: Inter-Iteration State Propagation

## Question

How is state preserved and transformed between loop iterations, particularly regarding gap tracking and resolution outcomes?

## Context

FR1.5.1.1 describes reconciliation as "evaluate rules, collect gaps, execute resolutions, repeat." This implies state must flow between iterations, but the mechanism is unspecified:

**What state crosses iteration boundaries?**
- All previously-seen gap identities (for deduplication)?
- Only currently-active (unresolved) gaps?
- Resolution outcomes from previous iterations?
- Accumulated errors/warnings?

**Transformation questions:**
- Do we create fresh gap instances each iteration, or mutate existing ones?
- When rules re-detect a gap that was resolved, is the old instance updated or is this treated as a new gap?
- How do we efficiently diff "gaps from iteration N" vs "gaps from iteration N+1"?

**Effect-TS patterns:**
- Should we use `Effect.iterate` with an accumulator?
- Should we use `Ref` for mutable iteration state?
- Should state transformation be explicit (return new state) or implicit (update refs)?

This blocks implementation because the choice affects all loop logic, testability, and whether iteration state is inspectable (NFR2.4.4).

Related requirements: FR1.5.1.1-4, FR1.5.3.1-4, NFR2.4.4
