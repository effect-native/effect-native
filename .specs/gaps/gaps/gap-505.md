# Gap: State Observation and Progress Reporting

## Question

How should internal loop state be exposed for observability, debugging, and progress reporting during execution?

## Context

Multiple requirements imply state observation needs:
- NFR2.4.4: "Engine.Test layer shall allow inspection of internal state after reconciliation"
- NFR2.3.2: "log sufficient context for debugging without excessive verbosity"
- FR1.5.3.1-4: Final result must include iteration count, all gap states, termination reason

**Unspecified observation requirements:**
1. **During execution**: Can callers observe progress mid-loop? (e.g., "iteration 3/10, 5 gaps remaining")
2. **State snapshots**: Should we support point-in-time state snapshots for debugging?
3. **Event stream**: Should state changes emit events (gap detected, gap resolved, iteration complete)?
4. **Test hooks**: Beyond final state inspection, can tests observe intermediate states?

**Effect-TS patterns available:**
- `Stream` for event emission
- `SubscriptionRef` for observable state
- Logging spans for structured observability
- Return accumulated history in result

**Tradeoffs:**
- Rich observability adds complexity and potential performance cost
- Minimal observability makes debugging and testing harder
- Event streams vs polling have different composition characteristics

Related requirements: NFR2.3.2, NFR2.4.4, FR1.5.3.1-4, FR1.2.3.3
