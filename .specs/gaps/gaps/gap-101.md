# Gap: Gap Identity Field Declaration Mechanism

## Question

How should gap types declare which fields contribute to identity computation vs. which are metadata?

## Context

FR1.2.2.1-3 specify that gap identity is computed from `_tag` plus "identity-contributing fields", and that gap types may optionally declare explicit identity fields. However, the mechanism for this declaration is unspecified.

Options include:
1. A static property on the class (e.g., `static identityFields = ['userId', 'resourceId'] as const`)
2. A schema annotation using Effect Schema
3. A separate registry mapping `_tag` to identity field names
4. A marker type/interface (e.g., `GapIdentity<['userId', 'resourceId']>`)

This affects:
- Type safety of identity computation
- How gaps are defined by users
- Whether identity configuration can be validated at registration time
- Serialization/deserialization of identity

Related requirements: FR1.2.2.1, FR1.2.2.2, FR1.2.2.3, FR1.2.2.5
