# v4 Migration: Definition of Done

## Success Criterion

```bash
bun run ok
```

`bun run ok` exits `0` and verifies deploy readiness for the `v4` line.

---

## Gate 1: Deploy Readiness Command

```bash
bun run ok
```

Pass condition: command exits `0` in CI and local dev.

---

## Gate 2: All Tests Green

```bash
bun run test
```

Pass condition: zero failing tests across all workspaces.

---

## Gate 3: Workspace Path Is `packages/`

```bash
bun -e 'const p=require("./package.json");const ws=Array.isArray(p.workspaces)?p.workspaces:(p.workspaces?.packages??[]);const hasPackages=ws.some((w)=>String(w).startsWith("packages/"));const hasPackagesNative=ws.some((w)=>String(w).includes("packages-native"));if(!hasPackages||hasPackagesNative)process.exit(1)'
```

Pass condition: workspace globs include `packages/` and do not reference `packages-native/`.

---

## Gate 4: Packages Support Effect v4 Beta

```bash
bun run check && bun run test
```

Pass condition: all packages compile and test successfully against current Effect v4 beta dependencies.

---

## Gate 5: Release Channel From `origin/v4` Is Beta

```bash
git merge-base --is-ancestor origin/v4 HEAD
rg -n "changeset publish --tag beta" .github/workflows/release.yml
```

Pass condition: release commits are based on `origin/v4` and release workflow publishes with the npm `beta` dist-tag.

---

## Gate 6: Effect-Named Package Versions Are v4

```bash
bun -e 'const fs=require("fs");const dirs=fs.readdirSync("packages",{withFileTypes:true}).filter((d)=>d.isDirectory());for(const d of dirs){const f=`packages/${d.name}/package.json`;if(!fs.existsSync(f))continue;const j=JSON.parse(fs.readFileSync(f,"utf8"));const name=String(j.name||"");if(name.includes("effect")&&j.private!==true){if(!String(j.version||"").startsWith("4."))process.exit(1)}}'
```

Pass condition: every publishable package whose name contains `effect` has a `4.x.x` version so installs by `@effect-native/*@^4` and `@effect-native/*@beta` resolve to the v4 line.

---

## Infrastructure State at Done

- package manager is Bun (`bun.lock` tracked)
- test runner is Bun (`bun test` / `bun run test`)
- canonical verification command is `bun run ok`
- workspace root is `packages/`
- release branch is `v4` and npm channel/tag is `beta`
